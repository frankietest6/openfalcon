Counts pending requests for the same song across all viewers. Set to 0 for unlimited.

Tiebreak

When enabled, a strict tie at the top of the vote count triggers a tiebreak round — viewers see an urgent banner listing the tied songs and have a limited window to vote again. If the timer expires without a clear winner, all votes for that round are dumped and FPP plays its scheduled next song.

Disabled by default; without this, ties are broken by "earliest first vote wins."

— typical: 30–90s

Audio enabled

Master switch for ShowPilot's built-in audio. Turn off if you deliver audio externally (PulseMesh, FM transmitter, Icecast) and don't want ShowPilot to host audio at all. When disabled: the listen-on-phone button is hidden from viewers, audio streaming endpoints are turned off, and the FPP plugin skips uploading audio files entirely (saves disk space and sync time). Other audio settings below have no effect when this is off.

Audio distance gate (copyright safety)

When enabled, viewers must be physically near your show to listen. Tapping the 🎧 button forces a fresh GPS check at click time — cached location from earlier in the day or a different location won't bypass it. Location is also re-verified every 15 minutes during playback, so a viewer who walks away will be cut off automatically. Recommended for shows playing licensed/copyrighted music. Leave disabled if you only play original content or have streaming rights. Uses the show coordinates set in Interaction Safeguards.

— typically smaller than the interaction radius

Audio sync offset

Adjusts when audio starts playing on listeners' phones relative to the light show. If phones sound ahead of the lights, use a positive value (delays audio). If phones sound behind, use a negative value. Typical range is 0–2000ms. Changes take effect on the next song.

— positive = delay audio (phones were ahead); negative = advance audio (phones were behind)

Audio daemon

ShowPilot connects to the audio daemon running on your FPP Pi to relay live audio to all listeners simultaneously. The daemon is automatically started by the ShowPilot FPP plugin on port 8090. Only change this if you configured a different port in the plugin.

— default 8090, must match ShowPilot plugin setting

External audio access

Listeners on cellular or away from your home network can't reach your local IP. Set your public domain so the player automatically falls back to it for external listeners. Local listeners still use the direct path for best performance.

The full URL where this ShowPilot instance is publicly reachable. Leave blank to disable external access.

PSA injection

Periodically replaces a played sequence with a "PSA" sequence — useful for tip-jar reminders, FM frequency callouts, safety messages, sponsor mentions. Mark a sequence as a PSA in the Sequences tab. The least-recently-played PSA is chosen each time.

Interactions mode. The counter ticks up each time a viewer votes or makes a jukebox request. After N of those, the next viewer-driven play is replaced with a PSA. Best for shows where most plays are viewer-driven. If nobody is voting or requesting, PSAs never fire.
Sequences mode. The counter ticks up each time FPP plays a sequence (PSAs themselves don't count). After N sequences, the next plugin poll receives a PSA to play. Fires regardless of viewer activity — best for schedule-driven shows that want regular sponsor or safety reminders even when no one is voting.

IP blocking

View-source deterrent (template protection)

When enabled, the viewer page goes through several layers of casual deterrence:

Ctrl+U, F12, and Ctrl+Shift+I/J keyboard shortcuts are all blocked — pressing them does nothing. Right-click on the page is disabled, blocking the context menu's "View Page Source" and "Inspect" options. If someone bypasses those (e.g. by typing view-source: in the URL bar), the source they see is a friendly "nothing to see here" message instead of your real template. The real page still renders normally for visitors — they see the show as usual, just without easy access to the source. Useful if you've purchased a viewer template from a creator and don't want casual visitors copying it.

Important: this is a deterrent, not real protection. Determined users can still open DevTools through the browser menu (⋮ → More tools → Developer tools), where keyboard shortcut blocking has no effect. From there they can inspect the live DOM or recover the page from network requests. Don't rely on this to enforce a license — use it to discourage casual snooping.

Public Access (Cloudflare Tunnel)

Loading…

Share your show with anyone on the internet — no port forwarding, no static IP, no domain rewiring on your router. You bring your own free Cloudflare account; ShowPilot runs the tunnel for you.

—
—

Step 1 — install the tunnel runtime. This downloads the cloudflared binary from Cloudflare's official GitHub releases into ShowPilot's data directory. Takes about 30 seconds on most networks.

Step 2 — connect a Cloudflare tunnel. You'll need a free Cloudflare account. Show me how →

  1. Sign in (or sign up) at one.dash.cloudflare.com. Add a domain to your Cloudflare account if you don't have one yet — Cloudflare also sells domains for ~$10/year if you need one just for this.
  2. Go to Networks → Tunnels and click Create a tunnel. Choose the Cloudflared connector type and name it anything (e.g. my-light-show).
  3. On the next screen, copy the token from the install command. The token is the long string after --token. Don't run any commands — just copy the token.
  4. Click Next, then on the Public Hostname page: pick a subdomain (e.g. show), pick your domain, and set Service to HTTP with URL 127.0.0.1:3100 (or whatever port your ShowPilot is on). Save.
  5. Paste the token below and click Save Token. Your show is now live at the hostname you configured.

Recommended: protect your /admin path with Cloudflare Access (free for up to 50 users) so only you can reach the admin panel. Your viewers don't need it; only you do.

Treat this like a password. Anyone with the token can connect to your tunnel.

Your tunnel is configured. The public URL is whatever hostname you set up in your Cloudflare dashboard.
